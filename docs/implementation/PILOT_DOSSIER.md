# AfyaIntel Pilot Dossier

## Purpose

Prepare AfyaIntel for responsible review by a facility, district health team, university reviewer, or digital-health partner.

## Pilot readiness stages

| Horizon | Evidence required |
|---|---|
| 0–3 months | Working offline prototype, synthetic dataset, dashboard mockups, data dictionary |
| 4–6 months | Single-clinic demonstration, usability feedback, sync logs, stock lookup tests, clinical review checklist |
| 7–12 months | Controlled pilot in 3–5 facilities, NLP F1 report, forecasting error report, SUS score, FHIR validation |
| 13–24 months | District-level dashboard, data-quality audit, governance report, sustainability model |

## Required approvals before real data

- Facility authorization
- Ethical review where applicable
- Data protection impact assessment
- Clinical governance review
- Role-based access configuration
- Staff training and incident-response procedure

## Pilot metrics

- NLP precision, recall, and F1-score
- Forecasting MAE, RMSE, WAPE, and stock-out lead time
- Offline sync success rate and queue recovery time
- System Usability Scale score
- Human review completion rate
- FHIR validation pass rate
