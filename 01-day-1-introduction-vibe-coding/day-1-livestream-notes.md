## Main Lessons

- AI coding agents are already part of normal professional development workflows.
- Specification and verification become bottlenecks as implementation becomes faster.
- Agent skills and dynamic context help close the final 20% of complex work.
- Verification loops, sub-agent evaluation, and human review are essential for production quality.
- Long-running agents are useful when progress can be checked continuously.
- Risks include loss of human codebase knowledge, unclear accountability, security gaps, and missed innovation.

## Impressive, Useful, Sustainable

A strong capstone should move through three stages:

1. **Impressive:** the demonstration works and attracts attention.
2. **Useful:** it solves a real, repeatable user problem.
3. **Sustainable:** it is safe, maintainable, affordable, and scalable.

AfyaIntel's local-first redesign moves it beyond an impressive API demo toward a more useful and sustainable prototype.
